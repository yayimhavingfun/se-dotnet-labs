## Подробное описание лабораторных работ

### Лабораторная 1: Симулятор транспортного маршрута маглев-поездов

- **Отрабатываемые темы**: инкапсуляция, сокрытие данных, композиция, полиморфизм  
- **Основные задачи**: разработка объектной модели поездов и маршрутов (обычные пути, силовые магнитные пути, станции), итеративный расчёт движения, обработка лимитов скорости, финальная остановка  

### Лабораторная 2: Корпоративная система распределения сообщений

- **Отрабатываемые темы**: основные принципы ООП, GRASP, SOLID, структурные паттерны проектирования (Decorator, Composite, Adapter, Proxy и др.), mocking для тестирования  
- **Основные задачи**:  
  - Разработка модели сообщений (заголовок, тело, уровень важности)  
  - Форматтеры сообщений (консольный и файловый) с Markdown-форматированием  
  - Архиваторы: форматирующий (с использованием форматтера) и in-memory  
  - Системы оповещений: текстовая и звуковая (Console.Beep)  
  - Пользователи как конечные точки с отслеживанием статуса сообщений («не прочитано» / «прочитано»)  
  - Топики для групповой рассылки  
  - Разнообразные адресаты:  
    - Адресат-пользователь  
    - Адресат-архиватор  
    - Адресат-группа (композитный паттерн)  
    - Адресат-система оповещений (с проверкой на ключевые слова)  
  - Фильтрация сообщений по уровню важности  
  - Логгирование действий адресатов (тестируемое)  

### Лабораторная 3: Симулятор сражений фэнтези-существ

- **Отрабатываемые темы**: основные концепции ООП, SOLID (особенно OCP, DIP), порождающие паттерны (Builder, Factory Method / Abstract Factory для конфигурации существ и каталога), структурные паттерны (Decorator для модификаторов, возможно Composite/Proxy)  
- **Основные задачи**:  
  - Модель существ: базовые показатели (атака/здоровье), атака другого существа, получение урона, временные изменения характеристик (только на время боя или на столе игрока)  
  - Каталог существ: фабрики/билдеры для создания преднастроенных существ без изменения оригинала  
  - Модификаторы: наложение нескольких (в т.ч. одинаковых), комбинирование эффектов — Магический щит (однократный блок урона), Мастерство атаки (двойная атака при выживании)  
  - Заклинания: постоянные улучшения на столе игрока — Зелье силы (+5 атака), Зелье выносливости (+5 здоровье), Амулет защиты (Магический щит), Волшебное зеркало (swap атака ↔ здоровье)  
  - Столы игроков: до 7 существ, независимые изменения (не влияют на каталог или другие столы)  
  - Бой: поочерёдные атаки случайных существ оппонента (только живые и с >0 атаки), победа/ничья по правилам (нет атакующих/атакуемых → исходы), без встречного урона  

### Лабораторная 4: Консольный файловый менеджер с расширяемыми командами

- **Отрабатываемые темы**: SOLID (особенно SRP, OCP, DIP), поведенческие паттерны (Command для команд, Interpreter/Chain of Responsibility для парсинга), структурные (Facade/Adapter для абстракции ФС, Decorator для расширения параметров), порождающие (Factory для создания команд/провайдеров ФС)  
- **Основные задачи**:  
  - Навигация по unix-подобным путям (абсолютные/относительные, `.`, `..`, без выхода за пределы пути подключения)  
  - Команды: `connect [Address] [-m Mode]`, `disconnect`, `tree goto [Path]`, `tree list {-d Depth}`, `file show [Path] {-m Mode}`, `file move/copy [Source] [Dest]`, `file delete [Path]`, `file rename [Path] [Name]`  
  - Вывод дерева каталога с параметризуемыми символами (папка/файл, отступы) и глубиной (по умолчанию 1)  
  - Поддержка только локальной ФС (режим `local`), возможность переключения (connect/disconnect)  
  - Консольный интерфейс: цикл ввода команд после вывода результата
 
### Лабораторная 5: Система банкомата (гексагональная / чистая архитектура)

- **Отрабатываемые темы**: многослойные архитектуры (гексагональная / чистая / луковая), Dependency Inversion, Dependency Injection (Microsoft.Extensions.DependencyInjection), модульная регистрация сервисов, богатая доменная модель (опционально), тестирование бизнес-логики с моками  
- **Основные задачи**:  
  - Создание и управление сессиями:  
    - Пользовательская (по номеру счёта + PIN)  
    - Админская (по системному паролю)  
    - Возврат уникального ключа сессии (Guid)  
  - Операции со счётом (только в рамках активной сессии):  
    - Просмотр баланса  
    - Снятие денег (проверка достаточности)  
    - Пополнение  
    - Просмотр истории операций  
  - Каждая операция со счётом фиксируется в истории  
  - Web API на ASP.NET Core контроллерах
